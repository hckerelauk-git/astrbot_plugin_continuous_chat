# 持续对话 (astrbot_plugin_continuous_chat)

让 AstrBot 支持**关键词唤醒**、**正则唤醒**，唤醒后可进入**持续对话模式**，群内所有消息自动触发 AI，无需重复喊唤醒词。

[English](./README_en.md)

## 功能

### 唤醒方式

1. **关键词唤醒**：消息中包含配置的关键词即触发唤醒
   - 例如配置 `["雪酱", "小助手"]`，发送"雪酱在吗"即可唤醒
2. **正则唤醒**：消息匹配正则表达式时触发唤醒
   - 使用 `re.search` 模式，直接写正则，无需 `.*`
   - 例如配置 `["^雪酱", "在吗[?？]"]`，匹配以"雪酱"开头或含"在吗?"的消息

### 持续对话模式

- 唤醒后自动进入持续对话状态
- 持续期间**本群所有消息**都会触发 AI，无需重复喊唤醒词
- 可配置持续时长（默认 60 秒），超时自动退出
- 支持回复后重置计时：`reset_on_reply`
- 也可通过指令手动控制：

| 指令 | 说明 |
|------|------|
| `/wbegin` | 手动进入持续对话状态 |
| `/wexit` | 手动退出持续对话状态 |
| `/wgid` | 查看当前群聊 ID（用于配置白名单） |

### 其他特性

- **群聊白名单**：可限制只有特定群聊启用此插件
- **私聊唤醒**：私聊中也支持关键词/正则唤醒（私聊本身是持续对话，不需要持续模式）
- **后台自动清理**：超时群聊自动清理，无需手动管理

## 安装

### 方式一：通过 AstrBot WebUI 安装（推荐）

1. 前往 [AstrBot 插件市场](https://plugins.astrbot.app)
2. 搜索"持续对话"并安装

### 方式二：手动安装

1. 克隆本仓库到 AstrBot 插件目录：

```bash
cd AstrBot/data/plugins
git clone https://github.com/hckerelauk-git/astrbot_plugin_continuous_chat
```

2. 重启 AstrBot，或在 WebUI 插件管理页点击"重载插件"

## 配置

在 AstrBot WebUI -> 插件管理 -> 持续对话 中配置：

| 配置项 | 类型 | 说明 |
|--------|------|------|
| 唤醒关键词 | list | 触发唤醒的关键词，留空则仅用正则 |
| 唤醒正则表达式 | list | 触发唤醒的正则（re.search 模式） |
| 群聊白名单 | list | 留空则不限制，为空则不启用白名单功能 |
| 启用私聊唤醒 | bool | 开启后私聊也支持关键词/正则唤醒 |
| 启用持续唤醒 | bool | 开启后被唤醒的群聊进入持续模式 |
| 持续时长（秒） | int | 持续对话的最长时间，默认 60 |
| 回复后重置计时 | bool | 每次 AI 回复后重置持续时间 |

## 使用示例

### 基础唤醒

配置唤醒关键词为 `["雪酱"]`：
- 发送"雪酱你好" → 触发 AI 回复
- 发送"雪酱在吗" → 触发 AI 回复

### 正则唤醒

配置唤醒正则为 `["^help", "求助[?？]"]`：
- 发送"help me" → 触发（因为匹配 `^help`，但实际是 `re.search`，所以不是必须从头匹配）
- 发送"求助？" → 触发

### 持续对话

1. 配置启用"持续唤醒"，时长 60 秒
2. 发送"雪酱在吗" → 触发并进入持续对话
3. 接下来 60 秒内，本群**所有消息**都会触发 AI，无需再喊唤醒词
4. 每条 AI 回复重置计时，最后一条消息后 60 秒自动退出

### 白名单

1. 发送 `/wgid` 获取群聊 ID
2. 将 ID 添加到"群聊白名单"配置项
3. 只有白名单内的群聊才会触发插件功能

## 指令列表

| 指令 | 适用场景 | 说明 |
|------|----------|------|
| `/wbegin` | 群聊 | 手动进入持续对话状态 |
| `/wexit` | 群聊 | 手动退出持续对话状态 |
| `/wgid` | 群聊 | 查看当前群聊 ID |

## 技术细节

- 基于 AstrBot 插件系统开发
- 使用 `event.is_at_or_wake_command = True` 触发 AI 响应
- 后台 asyncio 任务自动清理超时的群聊状态
- 正则表达式预编译，性能更好
- 完整的 `terminate()` 清理逻辑，卸载/重载插件时无残留
